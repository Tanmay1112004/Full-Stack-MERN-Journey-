Mobile-First Principle

How Mobile-First Works
‚úÖ Correct (Mobile-First)
<!-- Start with mobile --> <div class="col-12 col-md-6 col-lg-4"> Content </div> <!-- Inheritance: --> Mobile (<576px): col-12 (Full) Tablet (‚â•768px): col-md-6 (Half) Desktop (‚â•992px): col-lg-4 (Third)
‚ùå Incorrect (Redundant)
<!-- Unnecessary repetition --> <div class="col-12 col-sm-12 col-md-12 col-lg-12"> Content </div> <!-- Just use: --> <div class="col-12"> Content </div>
üìå Key Rule:

Define styles for mobile first. Larger screens inherit mobile styles unless overridden. This means you only need to specify what changes at larger breakpoints.

Quick Reference Rules

Do's ‚úÖ
  • Start with mobile-first design
  • Use col- for mobile
  • Add col-md-, col-lg- for larger screens
  • Keep column total ‚â§ 12 per row
  • Let columns wrap naturally
  • Test on multiple devices
Don'ts ‚ùå
  • Don't repeat same class for all breakpoints
  • Don't place content directly in rows
  • Don't forget container around rows
  • Don't ignore column wrapping
  • Don't use fixed pixel values in grid
  • Don't skip responsive testing
