Website Integration Quiz & Reference

Question 1: Where should the CCBP UI Kit Script Code be placed?

<body> <!-- Section content here --> <!-- Where to place CCBP UI Kit? --> </body>

Answer: Just before the closing </body> tag

<body> <!-- All sections here --> <script type="text/javascript" src="https://d2clawv67efefq.cloudfront.net/ccbp-static-website/js/ccbp-ui-kit.js"> </script> </body>

Reason: The script needs to be loaded after all HTML elements are parsed.

Question 2: Which HTML attribute is used to uniquely identify elements?

Answer: id attribute

<!-- Correct --> <div id="sectionHome">...</div> <div id="sectionAbout">...</div> <!-- Each id must be unique -->

Note: For CCBP UI Kit, IDs must start with "section" prefix.

Question 3: When integrating with CCBP UI Kit, section container IDs must start with what prefix?

Answer: section prefix

<!-- WILL WORK --> <div id="sectionHome">...</div> <div id="sectionFavouritePlaces">...</div> <!-- WILL NOT WORK --> <div id="homeSection">...</div> <div id="favorites">...</div>

Important: The prefix must be lowercase "section" followed by name.

Question 4: Choose the correct syntax for onclick attribute:

A) onclick= "display(section2)" B) onclick= display (section2) C) onclick="display('section2')"

Answer: C) onclick="display('section2')"

Rules:

  • Double quotes around the entire attribute value
  • Single quotes inside the display() function
  • No spaces around equals sign
  • Parentheses must be included

Question 5: The onclick event occurs when an HTML element is clicked by the user.

Answer: True

<!-- Example --> <button onclick="display('sectionAbout')"> Click me to go to About section </button> <!-- Also works on other elements --> <div onclick="display('sectionHome')"> Click this div to go home </div>

The onclick attribute can be added to any HTML element, not just buttons.

  • Only one section visible
  • Use back buttons in each section
