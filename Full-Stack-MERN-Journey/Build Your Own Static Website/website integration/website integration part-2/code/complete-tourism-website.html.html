Introduction to HTML Lists

HTML lists are used to group related items together. They make content more readable and organized.

Real-world Examples:

Unordered List Examples:

  • Shopping list items
  • Features of a product
  • List of hobbies
  • Ingredients in a recipe

Ordered List Examples:

  1. Step-by-step instructions
  2. Ranking of items
  3. Chronological events
  4. Priority tasks

1. Unordered Lists (<ul>)

Used for items with no specific order or sequence.

<!-- Basic Unordered List --> <ul> <li>First item</li> <li>Second item</li> <li>Third item</li> </ul>

List Style Types:

Disc (Default)

  • Item 1
  • Item 2
  • Item 3
list-style-type: disc;

Circle

  • Item 1
  • Item 2
  • Item 3
list-style-type: circle;

Square

  • Item 1
  • Item 2
  • Item 3
list-style-type: square;

None

  • Item 1
  • Item 2
  • Item 3
list-style-type: none;

2. Ordered Lists (<ol>)

Used for items with specific order or sequence.

<!-- Basic Ordered List --> <ol> <li>Step 1: Prepare ingredients</li> <li>Step 2: Mix everything</li> <li>Step 3: Bake for 30 minutes</li> </ol>

Numbering Styles:

Decimal (Default)

  1. First item
  2. Second item
  3. Third item
list-style-type: decimal;

Upper Roman

  1. First item
  2. Second item
  3. Third item
list-style-type: upper-roman;

Lower Alpha

  1. First item
  2. Second item
  3. Third item
list-style-type: lower-alpha;

Upper Alpha

  1. First item
  2. Second item
  3. Third item
list-style-type: upper-alpha;

5. Quick Quiz

Question 1: Which HTML tag starts an unordered list?

Answer: <ul>

Question 2: Which HTML tag starts an ordered list?

Answer: <ol>

Question 3: What CSS property styles list markers?

Answer: list-style-type

Question 4: How do ordered and unordered lists differ?

Answer: Ordered lists show sequence (1, 2, 3...), unordered lists show items without specific order (bullets).

6. Best Practices

  • Use <ul> for navigation menus
  • Use <ol> for step-by-step instructions
  • Always close your <li> tags
  • Use CSS to style lists instead of inline styles
  • Nest lists properly for hierarchical data
  • Use semantic list types for accessibility
