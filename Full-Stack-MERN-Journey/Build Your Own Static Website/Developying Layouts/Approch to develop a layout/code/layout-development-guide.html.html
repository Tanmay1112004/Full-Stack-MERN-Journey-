Layout Development Guide

1. HTML Image vs CSS Background Image

HTML Image

Sale!

Use when:

  • Image is content (product, profile photo)
  • Needs alt text for accessibility
  • Users might want to save it

CSS Background Image

Content Over Image

Use when:

  • Content needs to overlay the image
  • Image is decorative/background
  • Need more control over image display
<!-- HTML Image --> <img src="image.jpg" alt="Description" class="img-fluid"> <!-- CSS Background Image --> <div class="hero-section"> <h1>Content over image</h1> </div> <style> .hero-section { background-image: url('image.jpg'); background-size: cover; background-position: center; } </style>

2. Margin vs Padding

Margin Only
Content

Space outside the element

Padding Only
Content

Space inside the element

Margin + Padding
Content

Both outside and inside space

<style> /* Padding: Space INSIDE, affects background */ .element { padding: 20px; background-color: blue; /* This covers padding area */ } /* Margin: Space OUTSIDE, doesn't affect background */ .element { margin: 15px; background-color: red; /* This does NOT cover margin area */ } /* Combined Example */ .button { padding: 12px 24px; /* Space inside button */ margin: 10px; /* Space between buttons */ background-color: #007bff; color: white; } </style>

3. Step-by-Step Layout Development Process

Step 1: Divide Layout into Sections

Break down the design into logical sections (Header, Hero, Features, Footer).

Step 2: Choose Image Type

HTML Image if:
  • No overlay content
  • Content image
  • Needs alt text
CSS Background if:
  • Content overlays image
  • Decorative background
  • Hero sections

Step 3: Apply Spacing

Use padding for internal spacing, margin for external spacing between elements.

Step 4: Use Flexbox for Alignment

Apply Bootstrap flex classes for proper alignment and responsive behavior.
